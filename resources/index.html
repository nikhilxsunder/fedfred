  • Special Notes: Parameter Conversion and Nested Classes (FedFred 2.0+)

Topics include: - Parameter auto-conversion (e.g., datetime.datetime, list handling), - Nested class structure (fedfred.clients.FredAPI.AsyncAPI, fedfred.clients.FredAPI.MapsAPI, fedfred.clients.FredAPI.AsyncAPI.AsyncMapsAPI), - DataFrame backend configuration options (Pandas, Polars, Dask).

For a high-level overview of how FedFred is structured internally, see Internal Architecture.
